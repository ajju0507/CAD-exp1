2. Create an S3 Bucket
Open S3 from the AWS console.
Click "Create bucket".
Enter a unique bucket name (e.g., my-static-website).
Choose a region (select the closest to you).
Disable Block all public access and acknowledge the warning.
Click "Create bucket".

3. Enable Static Website Hosting
Open the created bucket.
Go to the Properties tab.
Scroll to Static website hosting → Click Edit.
Choose "Enable".
Set Index document to index.html (or your main file).
(Optional) Set an Error document (e.g., error.html).
Click Save.

4. Upload Website Files
Go to the Objects tab.
Click Upload → Add index.html, CSS, JS, and images.
Click Upload.

index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Static Website</title>
    <style>
        body { text-align: center; font-family: Arial, sans-serif; margin: 50px; }
        h1 { color: #4CAF50; }
    </style>
</head>
<body>
    <h1>Welcome to My Static Website!</h1>
    <p>This website is hosted on AWS S3.</p>
</body>
</html>


error.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>404 - Page Not Found</title>

<body>
    <h1>404 - Page Not Found</h1>
    <p>Oops! The page you are looking for does not exist.</p>

</html>


5. Set Bucket Permissions for Public Access
Go to the Permissions tab.
Scroll to Bucket Policy → Click Edit.
Paste this JSON policy (replace <your-bucket-name> with your actual bucket name):

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::<your-bucket-name>/*"
    }
  ]
}

Click Save changes.

6. Test the Deployment
Go to Properties → Scroll to Static website hosting.
Copy the Bucket Website URL.
Open it in a browser to view your website.
